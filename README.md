# Bronia Portfolio Website

## Project Overview / Documentation

This project is a personal portfolio website presenting the professional profile of **Irvin C. Bronia**, a Bachelor of Science in Information Technology student specializing in **Web Development** at **Camarines Norte State College**.

---

## Technologies Used

- **General:** HTML, CSS, JavaScript  
- **APIs:** OpenWeatherMap, Leaflet, Advice API, EmailJS

---

## Features

### 1. Portfolio Sections

- **Home / Introduction:**  
  Displays full name, professional summary, course, specialization, profile photo, weather checker (API), Leaflet map (API), and random quote generator (API).  

- **About Me:**  
  Provides personal background, educational history, and career goals.  

- **Skills / Technologies:**  
  Lists programming languages, tools, frameworks, and software proficiency.  

- **Projects:**  
  Showcases projects including Figma prototypes, video editing, and profile websites with demos or embedded previews.  

- **Contact:**  
  Contact form integrated with EmailJS for sending messages directly.  

- **Donate / Pay:**  
  Mock donation/payment simulation with multiple payment options and dynamic input fields.  

---

### 2. API Integrations

The website integrates the following APIs:

1. **OpenWeatherMap API** – Displays current weather based on the user’s city input, including temperature, weather description, and dynamic map positioning.  
2. **Leaflet API** – Interactive map visualization showing the city location alongside weather data.  
3. **Advice API (api.adviceslip.com)** – Generates dynamic “Quotes to live by.”  
4. **EmailJS** – Handles contact form submissions with email delivery without requiring a backend.  

> These APIs are directly integrated using JavaScript.

---

### 3. Transaction Features

1. **Contact Form**  
   - Users can submit their name, email, and message.  
   - JavaScript validates required fields before submission.  
   - Successful submissions display a confirmation message; errors prompt the user to complete missing fields.  

2. **Mock Donation / Payment Form**  
   - Users enter full name, email, amount, and select a payment method (GCash, PayPal, Card).  
   - Dynamic input fields appear based on the selected payment method.  
   - JavaScript validates all fields and simulates payment processing with a transaction ID.  
   - Displays success or error messages dynamically without real payment processing.  

---

### 4. How to Run the Project

**Via folder download**  
1. Clone or download the project folder.  
2. Open `index.html` in a web browser (Chrome, Edge, Firefox recommended).  
3. Ensure an active internet connection to allow API calls and map loading.  

**Via GitHub repository / Deployed site**  
1. Open the live site via the GitHub Pages link.  
2. Ensure an active internet connection.  
3. Navigate through tabs to view Home, About Me, Skills, Projects, Contact, and Donate/Pay sections.  
4. Interact with:  
   - Weather widget (enter a city and click "Get Weather")  
   - Quote generator (click "New Quote")  
   - Contact form (submit your name, email, and message)  
   - Mock donation/payment form (fill out fields and select payment method)
